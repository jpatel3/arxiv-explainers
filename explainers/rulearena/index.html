Can LLMs follow real-world rules correctly?

RuleArena is a benchmark testing whether LLMs can faithfully apply authentic, complex regulations — from airline baggage fees to the NBA CBA to US tax forms — to produce verifiable, correct answers. 95 rules. 816 problems. 6 frontier models. Even the best reasoning model solves only ~50% of the easiest problems.

How RuleArena works
The evaluation framework in 4 steps
1

Rules are sourced

Real policy documents (AA baggage policy, NBA CBA, IRS Form 1040 instructions) are extracted as authentic rule sets — up to 20K tokens long.

2

Problems are synthesized

Each domain generates test problems across 3 difficulty levels. Ground-truth answers come from a deterministic rule-based Python script (or expert annotation for NBA).

3

LLM is prompted

The model receives: task instruction + full rule set + user instance. It must produce a step-by-step computation ending in a structured answer line.

4

Dual-axis metrics

GPT-4o parses each free-text response into a structured rule-application trace. We score per-problem accuracy, recall, precision, and per-rule application correctness.

The four key findings

1. Low overall accuracy

Non-reasoning LLMs fail nearly all Level-1 problems in 0-shot mode. Even o1-preview solves only 54% of the easiest Airline problems. On hard levels, accuracy rarely exceeds 10%.

2. High precision (mostly)

When rules are unambiguous, LLMs pick the right ones (Airline/Tax precision = 100%). But NBA precision is only 50.4% — many rules are similar-but-distinct, confusing the model.

3. Low recall (missing rules)

LLMs miss necessary rules — they don't fully grasp the workflow. NBA recall is worst (30.8%); Tax recall is best (90%) because the 1040 has a clear linear structure.

4. High but imperfect application

Even when a rule IS recalled, occasional arithmetic or logic errors occur (~80% application correctness). Because the computation chains, one error zeros out the final answer.
